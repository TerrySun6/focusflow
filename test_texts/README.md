# 测试文本集说明

> 用于验证语义逻辑分析管线（PHATE + EMA + PELT + Leiden）的有效性

---

## 使用方法

将每个 `.txt` 文件的**绝对路径**粘贴到 Flask UI 的输入框中，运行分析。
观察以下指标：

| 指标 | 有逻辑文本预期 | 无逻辑文本预期 |
|------|:---:|:---:|
| PHATE 3D 形状 | 平滑曲线/直线 | 杂乱团块/分叉 |
| Tortuosity（弯曲度） | 较低 | 较高 |
| 连贯性均值 | 较高且稳定 | 较低且波动大 |
| PELT 分段数 | 少，边界清晰 | 多，碎片化 |
| 连贯性曲线 | 缓慢下降/平稳 | 剧烈震荡 |

## 预期分析结果对比

**最佳情况**（Tortuosity 低、连贯性高、分段少）：
- 有逻辑_说明_泡茶指南.txt（步骤清晰，逻辑链条最短最直）
- 有逻辑_论证_AI与教育.txt（论证结构完整）

**最差情况**（Tortuosity 高、连贯性低、分段多）：
- 无逻辑_跳跃_话题随机.txt（每句独立，0连贯）
- 无逻辑_拼凑_随机句子.txt（无任何语义关联）

**有趣案例**：
- 无逻辑_循环_车轱辘话.txt：连贯性可能不低（句子之间有表面关联），但语义信息量极低——这恰好是"假连贯"的典型
- 边界_混合_逐渐跑题.txt：前8句和后12句应该有明显的Tortuosity差异，预期PELT能在转折点附近切一刀
