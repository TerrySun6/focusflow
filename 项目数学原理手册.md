## 1. BGE-M3 / BERT Embedding

> 库：`sentence_transformers`, `BAAI/bge-m3`
> 作用：把句子变成 1024 维的数值向量

### 1.1 核心架构：Transformer 的双向注意力

BGE-M3 基于 **BERT 架构**（Bidirectional Encoder Representations from Transformers）。

每个句子经过 12~24 层 Transformer Encoder，每一层做两件事：

**（1）自注意力（Self-Attention）**

对于句子中的每个词，计算它和所有其他词（包括前后）的关联程度：

```
Attention(Q, K, V) = softmax(Q·Kᵀ / √d_k) × V
```

- Q（Query）：当前词在"查什么"
- K（Key）：其他词"有什么标签"
- V（Value）：其他词"携带什么信息"
- Q·Kᵀ：当前词和其他词的内积，表示"匹配程度"
- ÷√d_k：除以维度的平方根，防止内积太大导致 softmax 梯度消失
- softmax：把匹配分数变成 0-1 的概率分布
- ×V：按概率加权取其他词的信息

**（2）前馈网络（Feed-Forward）**

```
FFN(x) = ReLU(x·W₁ + b₁)·W₂ + b₂
```

对每个位置的向量做两次线性变换 + 一次非线性激活。

### 1.2 双向 vs 单向（为什么不用 GPT）

| | BERT | GPT |
|---|------|-----|
| 注意力方向 | 每个词看到**前后所有词** | 每个词只看到**前面的词** |
| 训练目标 | 完形填空（Masked LM） | 预测下一个词 |
| 适合做什么 | 理解（分类、相似度、聚类） | 生成（写文章、对话） |

我的任务是**理解句子语义**（不是生成文本），所以 BERT 类模型天然更合适。PPT 里提到最初用 GPT 类 Qwen-Encoder 效果不好，原因就在这里——单向注意力对句子级别的语义编码有信息缺失。

### 1.3 BGE-M3 的三个 M

- **M₁ (Multilingual)**：训练数据覆盖 100+ 语言，中英文混合文本无需切换模型
- **M₂ (Multi-granularity)**：支持最长 8192 token 输入，一个句子一个 embedding
- **M₃ (Multi-tasking)**：训练时同时优化检索、分类、聚类、语义相似度多个目标

### 1.4 Embedding 的本质

每个句子最终被压缩成一个 1024 维的向量：

```
我 爱 北京 天安门
  ↓   ↓   ↓    ↓
[0.12, -0.34, 0.78, ..., 0.05]  ← 1024 个数
```

这个向量的**方向**承载语义信息（相近含义 → 方向相近），**模长**受句子长度和高频词影响（不是语义信号）。

## 2. EMA 指数移动平均

> 自实现：`EMAnalyzer.run_ema()`
> 作用：平滑相邻句子的 embedding，减少措辞波动

### 2.1 数学定义

EMA（Exponential Moving Average）的核心思想：对时间序列上的每个点，用滑动窗口内的邻居做**加权平均**，权重按距离**指数衰减**。

对于位置 i，EMA 平滑后的值：

```
EMA[i] = Σⱼ wⱼ × embedding[i+j]
```

其中 j 从 -k 到 +k（k = window_size // 2），权重 wⱼ：

```
wⱼ = (1-α)^|j|
```

### 2.2 权重分布示例

α = 0.8, window = 5 时（β = 1-α = 0.2）：

```
位置:      i-2    i-1     i     i+1    i+2
权重:    0.027  0.135  0.676  0.135  0.027
         (β²)   (β¹)   (β⁰)   (β¹)   (β²)
```

中间句子（位置 i 自身）权重 0.676，最远邻居权重仅 0.027——25 倍差距。

### 2.3 边界处理

在文本开头（i < k）或结尾（i > n-k-1），窗口会超出范围。处理方式：
- 截断窗口到有效范围
- 重新归一化权重，使有效权重之和 = 1

### 2.4 EMA vs 普通滑动平均（SMA）

| | SMA | EMA |
|---|-----|-----|
| 权重分布 | 矩形窗（`1/window`） | 指数衰减（`(1-α)^|j|`） |
| 边界处理 | 窗口平移 | 截断 + 重新归一化 |
| 对局部突变的响应 | 迟钝 | 灵敏（离得近的权重更大） |

SMA 对所有窗口内邻居一视同仁，EMA 尊重"近邻更重要"的直觉。

### 2.5 α 参数的含义

```
α → 1  : β=0 → 只有自身有权重 → 几乎不平滑
α → 0  : β=1 → 窗口内均匀权重 → 平滑最重
α = 0.8: 默认值，中间 25 倍于最远
```

## 3. Centering 零均值中心化 & Anisotropy

> 自实现：`embeddings - np.mean(embeddings, axis=0)`
> 作用：消除大模型 embedding 的全局背景偏置

### 3.1 什么是各向异性偏置（Anisotropy Bias）

大语言模型（如 BERT、BGE-M3）在预训练时，所有 embedding 向量会**整体偏向某个方向**。这个方向对应的是"平均背景语义"——高频功能词、常见句式模式。

**后果**：即使两句完全无关的话，余弦相似度基线也不是 0，而是 ~0.85。

```
"紫色土豆在跳舞" vs "核反应堆结婚了"
→ 余弦相似度 ≈ 0.85  ← 不合理的高
```

所有数据点被一个巨大的"背景引力场"拽向同一角落，淹没了真正有意义的语义差异。

### 3.2 数学操作：Centering

```
centered_embeddings = embeddings - μ
```

其中 μ = 所有 embedding 向量的均值（1024 维向量）：

```
μ = (1/n) × Σᵢ embedding[i]
```

### 3.3 为什么有效

减去全局均值后：
- 均值向量变为 **零向量**
- 余弦相似度基线从 ~0.85 降到 ~**0.02**
- 局部语义突变信号被显著放大

**直觉类比**：所有点被同一个磁场拉着跑。Centering 就是把磁场关掉，让每个点回到自己本来的位置。

### 3.4 为什么不用 PCA 去偏

有些人用 PCA 去偏（如 BERT-flow），减掉前 K 个主成分。但 PCA 会**旋转和拉伸空间**，破坏原始 embedding 的相对距离。

Centering 只用均值减法——**不旋转、不拉伸**——所有相对几何关系原封不动保留。

## 4. 余弦相似度与连贯性指标

> 自实现：`EMAnalyzer.get_coherence()`
> 作用：衡量相邻句子的语义连贯程度

### 4.1 余弦相似度

两个向量 A 和 B 的余弦相似度：

```
cos(A, B) = (A · B) / (‖A‖ × ‖B‖)
```

其中：
- A·B：内积（两个向量逐元素相乘再求和）
- ‖A‖：A 的模长 = √(Σ aᵢ²)

**几何意义**：只看两个向量的**夹角**，不看**长度**。夹角越小 → 方向越一致 → 语义越相近。

范围：[-1, 1]，实际在语义向量中通常为 [0, 1]。

### 4.2 连贯性指标

连续文本中，相邻句子的语义应该是平滑过渡的。我定义连贯性为相邻句子对的余弦相似度：

```
coherence[i] = cos(normalized_emb[i], normalized_emb[i+1])
```

先 L2 归一化（每个向量除以自己的模长），再做内积：

```
normalized_emb[i] = emb[i] / ‖emb[i]‖
coherence[i] = normalized_emb[i] · normalized_emb[i+1]
```

### 4.3 连贯性在系统中的作用

- **平滑前后对比**：EMA 平滑前的平均连贯性 vs 平滑后的平均连贯性。如果 EMA 有效，平滑后连贯性应该上升。
- **分段信号**：连贯性急剧下降的位置 → 可能是语义转折点 → PELT 的输入信号之一。
- **异常检测**：单句连贯性明显低于全文平均 → 可能是跑题句或过渡句。

### 4.4 为什么归一化后余弦 = 内积

因为归一化后每个向量模长 = 1：

```
cos(A, B) = (A·B) / (1×1) = A·B
```

所以代码里直接用 `(norm[:-1] * norm[1:]).sum(dim=1)` 计算连贯性——等价于余弦相似度。

## 5. 降维方法对比

> 本章覆盖 PCA、t-SNE、PHATE、UMAP 四种降维方法。主线用 PHATE，其余三者用于对比实验。

---

### 5.1 PCA（主成分分析）

> 库：`sklearn.decomposition.PCA`
> 作用：找到数据方差最大的方向，投影到低维

**核心思想**：PCA 不做任何"流形学习"——它只是把高维数据做一个旋转变换，找到方差最大的几个方向（主成分），然后用这几个方向作为新的坐标轴。

**数学操作**：

对数据矩阵 X（n 个点，d 维），先做 Centering（减去均值），然后对协方差矩阵做特征分解：

```
C = (1/n) × XᵀX
C · v_i = λ_i · v_i
```

取前 k 个最大特征值 λ_i 对应的特征向量 v_i 组成投影矩阵 W_k，然后投影：

```
X_pca = X · W_k
```

**直觉**：把高维数据比作一团椭球形的面包屑，PCA 找到面包屑最长的几个轴线方向，沿着这些方向把面包屑压扁到 2D/3D。

**在我的项目中的表现**：

- ARI = 0.25（聚类保真度最低，几乎没用）
- Tortuosity = 875.9（超大，因为 PCA 只是线性投影，非线性语义结构完全丢失）
- 适合作为"地板基线"——比 PCA 差的方法不值得用

---

### 5.2 t-SNE（t-分布随机邻域嵌入）

> 库：`sklearn.manifold.TSNE`
> 作用：把高维邻居关系搬进低维，极度关注局部结构

**核心思想**：t-SNE 只关心"谁是谁的邻居"，完全不关心簇与簇之间的全局距离。先在高维用高斯核定义邻居概率，再在低维用 t 分布模仿这个概率，用 KL 散度让两者对齐。

**第 1 步：高维高斯概率**

对每个点 i，邻居 j 的条件概率：

```
p(j|i) = exp(-‖x_i - x_j‖² / 2σ_i²) / Σ_k exp(-‖x_i - x_k‖² / 2σ_i²)
```

σ_i 对每个点不同，由**困惑度（perplexity）**控制。Perplexity = 30 表示每个点约看 30 个邻居。

对称化：`p(i,j) = (p(j|i) + p(i|j)) / 2n`

**第 2 步：低维 t 分布相似度**

```
q(i,j) = (1 + ‖y_i - y_j‖²)⁻¹ / Σ_k≠l (1 + ‖y_k - y_l‖²)⁻¹
```

为什么用 t 分布？因为 t 分布尾巴重——低维中即使距离远，q 也不会太小。这让"假邻居"的排斥力温和，避免全局炸散。

**第 3 步：KL 散度优化**

```
Loss = Σ p(i,j) · log(p(i,j) / q(i,j))
```

- p 大 q 小 → 严重惩罚 → 拉近
- p 小 q 大 → 轻度惩罚 → 推开

结果：局部聚得很好，但簇间距离没有含义。

**在我的项目中的表现**：

- Trustworthiness = **1.00**（局部保真度完美）
- ARI = 0.45（不如 PHATE 的 0.47 和 UMAP 的 0.66）
- **簇与簇之间的距离没有语义含义**——不能用 Tortuosity 衡量全局逻辑
- 每次运行结果不同（随机初始化）

**适用场景**：可视化局部邻居关系。不适合需要全局距离的任务。

---

### 5.3 PHATE（热扩散势能嵌入）

> 库：`phate`
> 作用：保留全局势能距离，适合轨迹分析和跨文本距离比较

**核心思想来自热力学/扩散物理学**。

**第 1 步：局部亲和力**

```
A(i,j) = exp(-dist(i,j)² / σ²)
```

**第 2 步：扩散算子**

把亲和力矩阵按行归一化成马尔可夫转移矩阵 P，做 t 步随机游走：

```
P(i,j) = A(i,j) / Σₖ A(i,k)
P_t = P^t
```

t 用 Von Neumann 熵自动选择。

**第 3 步：势能距离**

```
D(i,j) = -log(P_t(i,j))
```

从 i 扩散到 j 的概率越大 → 势能距离越小 → 语义越近。

**第 4 步：MDS 降维**

用 Metric MDS 保持势能距离降到 3 维：

```
minimize: Σᵢⱼ (D(i,j) - ‖yᵢ - yⱼ‖)²
```

**在我的项目中的表现**：

- ARI = 0.47（仅次于 UMAP）
- Tortuosity：最低（轨迹最"直"，逻辑流线最清晰）
- **低维距离有意义**：PHATE 保留的是势能距离，跨文本可比（锚句法）
- 是系统主线降维方法

**关键参数**：

| 参数 | 含义 | 我的值 |
|------|------|--------|
| `n_components` | 降维目标维度 | 3 |
| `knn` | 最近邻数量 | 5 |
| `knn_dist` | 距离度量 | cosine |
| `mds` | MDS 方法 | metric（保留绝对距离） |
| `t` | 扩散时间 | auto（Von Neumann 熵自动选） |

---

### 5.4 UMAP（一致流形逼近与投影）

> 库：`umap`
> 作用：基于黎曼几何和模糊拓扑，保留全局拓扑结构

**核心思想**：假设高维数据躺在一个弯曲的流形上。先在每个局部区域用自适应半径画邻居图（数据密集处半径小、稀疏处半径大），再把同样的拓扑结构"搬"进低维。

**第 1 步：自适应模糊邻居图**

对每个点 i，找最近的非零邻居距离 ρ_i 作为"背景噪声"，计算模糊隶属度：

```
μ(i→j) = exp( -(dist(i,j) - ρ_i) / σ_i )
```

σ_i 通过二分搜索确定，使得 Σⱼ μ(i→j) ≈ log₂(K)。

对称化：`μ(i,j) = μ(i→j) + μ(j→i) - μ(i→j)·μ(j→i)`

**第 2 步：低维复刻邻居图**

用二元交叉熵（而不是 KL 散度）优化低维布局：

```
Loss = Σ 真邻居 log(1 / (1+a·‖yᵢ-yⱼ‖^(2b)))     ← 吸引力
     + Σ 假邻居 log(1 - 1/(1+a·‖yᵢ-yⱼ‖^(2b)))   ← 排斥力
```

a、b 由 `min_dist` 参数决定。相比 t-SNE 的 KL 散度，二元交叉熵对假邻居惩罚更温和——保留了更多全局结构。

**在我的项目中的表现**：

- ARI = **0.66**（四种方法最高，聚类效果最好）
- Trustworthiness = **1.00**（局部保真度满分）
- **但低维距离含义弱**：UMAP 保留的是拓扑连通性（谁连谁），不是度量距离（隔多远），不适合跨文本距离比较
- 轨迹较平缓——变化率比 PHATE 小，不适合 Tortuosity 分析

**关键参数**：

| 参数 | 含义 | 直觉 |
|------|------|------|
| `n_neighbors` | 邻居数 | 小→局部，大→全局。类似 PHATE 的 knn |
| `min_dist` | 低维最小距离 | 控制簇内紧密度 |

### 5.5 四方法对比总表

| | PCA | t-SNE | PHATE | UMAP |
|---|:---:|:---:|:---:|:---:|
| **理论基础** | 线性代数 | 概率 + KL 散度 | 扩散热力学 | 黎曼几何 + 拓扑 |
| **保留什么** | 方差 | 局部邻居 | **全局势能距离** | 全局拓扑结构 |
| **低维距离有意义吗** | 有（欧氏） | **没有** | **有（势能距离）** | 弱（拓扑距离） |
| **Trustworthiness** | 0.83 | **1.00** | 0.97 | **1.00** |
| **ARI** | 0.25 | 0.45 | 0.47 | **0.66** |
| **轨迹形状** | 散乱 | 团块状 | **花瓣形**（细节丰富） | 平缓（被熨平） |
| **每次稳定吗** | 稳定 | 不稳定 | 较稳定 | 不稳定 |
| **速度** | **快** | 慢 O(n²) | 慢 O(n²) | **快** O(n log n) |

> 数据基于《小王子》1331 句，BGE-M3 embedding 1024 维，EMA α=0.8 window=5 Centering 启用。

**选择理由**：我的任务是分析文本的"逻辑流线"——轨迹弯曲度（Tortuosity）和跨文本距离比较是关键需求。PHATE 是唯一同时满足"距离有意义"和"全局结构完整"的方法，因此作为主线。t-SNE 用于局部验证，PCA 作为地板基线，UMAP 作为速度与聚类的参考。

## 6. Leiden 聚类 & HDBSCAN

> 库：`leidenalg`, `igraph`, `hdbscan`
> 作用：对 PHATE 降维后的点进行聚类，识别语义簇

### 6.1 前置：构建相似度图

先从归一化后的 embedding 构建 KNN 图（代码中 k ≈ √n，限制在 6-18）：

- 每个点连向最近的 K 个邻居
- 边权重 = 余弦相似度（1 - 余弦距离）
- 去重（无向图）

### 6.2 Leiden 聚类

Leiden 是 Louvain 算法的改进版，用于**社区发现**（Community Detection）。

**核心目标：最大化模块度（Modularity）**

```
Q = (1/2m) × Σᵢⱼ [Aᵢⱼ - γ × (kᵢkⱼ / 2m)] × δ(cᵢ, cⱼ)
```

其中：
- Aᵢⱼ：节点 i 和 j 之间的实际边权重
- kᵢkⱼ/2m：随机图模型中 i 和 j 之间的期望边权重
- γ：resolution 参数（我的默认值 = 1.0）
- δ(cᵢ, cⱼ)：如果 i 和 j 在同一个簇 = 1，否则 = 0

**直觉**：
- 如果簇内边权重大于随机期望 → Q 上升 → 好聚类
- 如果簇内边权重小于随机期望 → Q 下降 → 差聚类

**Leiden 三步循环**：
1. 局部移动：逐个节点尝试移动到邻居簇，选 Q 增加最多的
2. 细化：在簇内做额外的局部优化，防止"差连接"
3. 聚合：把每个簇压缩成一个"超级节点"，回到步骤 1

**Resolution 参数（γ）**：
- γ ↑：惩罚大簇，产生更多小簇（高分辨率）
- γ ↓：奖励大簇，产生更少簇（低分辨率）
- γ = 1.0：标准模块度

### 6.3 HDBSCAN（回退方案）

如果 `leidenalg` 不可用，回退到 HDBSCAN。

HDBSCAN = 层次密度聚类。核心思想：
- 对数据做空间密度估计
- 找密度"山峰"作为簇中心
- 合并相邻的山峰
- 低密度区域的点标记为噪声

**关键参数**：`min_cluster_size` = 5（最小簇至少 5 个点）

### 6.4 Leiden 在管线中的作用

PELT 分段后，Leiden 聚类用于**合并相邻的同簇段**（代码第 280-288 行）。PELT 切出来的相邻段如果在 Leiden 聚类中属于同一个主题，就把它们合并——缓解 PELT 过分割问题。

## 7. PELT 变点检测

> 库：`ruptures`
> 作用：在 embedding 序列中找到语义转折的边界

### 7.1 什么是变点检测

给定一个时间序列（在这里是 embedding 序列），找出序列行为发生显著变化的时刻。

我们的序列：embedding[0], embedding[1], ..., embedding[n-1]

每个位置是一个 1024 维向量。变点 → 相邻句子的语义发生大转折。

### 7.2 PELT 的核心思想

PELT = **P**runed **E**xact **L**inear **T**ime

**第 1 步：定义代价函数**

对于序列的一段 [a, b)，计算它内部的"同质性"——段内所有点有多相似：

```
cost([a,b)) = Σₐᵇ ‖embedding[i] - mean([a,b))‖²
```

即段内每个点到段均值的 L2 距离平方和。如果一段内都是相同主题，cost 就小。

**第 2 步：定义优化目标**

找到一组切点 τ₁, τ₂, ..., τₖ，使得：

```
minimize: Σⱼ cost([τⱼ, τⱼ₊₁)) + β × (分段数)
```

其中 β 是 penalty（惩罚系数）。每多切一刀，代价加 β。

**第 3 步：动态规划求解**

```
F(t) = min_{0≤s<t} [F(s) + cost([s,t)) + β]
```

F(t) 表示从开头到位置 t 的最小总代价。递推关系：最后一个段从 s 到 t，之前的段的最优解是 F(s)。

直接 DP 是 O(n²)。PELT 通过**剪枝**降到 O(n)。

**第 4 步：剪枝（Pruning）**

PELT 的核心优化：如果位置 s 在位置 r 之后不可能成为任何最优解的一部分，就直接丢掉 s。

剪枝条件（两个不等式）确保丢掉的位置绝不会出现在最优解中。这使得实际运行时间为线性 O(n)。

### 7.3 我的 PELT 配置

**为什么 PELT 在原始 embedding 上运行？**

代码第 258 行注释：EMA 平滑会模糊边界，削弱变点检测信号。PELT 需要的是"突变"——在原始、不平滑的 embedding 上运行。

**Penalty 怎么算？**

```python
penalty = median_dist × pelt_penalty_multiplier
```

- `median_dist`：所有相邻句子的 L2 距离的中位数
- `multiplier`：默认 5.0，可调

这确保 penalty 是数据驱动的——不同文本的"正常波动"水平不同。

**模型选择**：

- 首选 `model="rbf"`（径向基函数核），假设段内数据服从高斯分布
- 如果失败（数据不符合假设），回退 `model="l2"`，penalty 降低到 0.1×

## 8. 评价指标体系

### 8.1 ARI（调整兰德指数）

> 库：`sklearn.metrics.adjusted_rand_score`

衡量两个聚类结果的一致性（高维聚类 vs 低维聚类）。

```
ARI = (RI - E[RI]) / (max(RI) - E[RI])
```

- RI（Rand Index）：随机抽两个点，它们在两次聚类中被分到一起/分开的概率
- E[RI]：随机聚类的 RI 期望值
- ARI 范围 [-1, 1]，1 = 完全一致，0 = 随机水平

**我的数据**：PHATE ARI = 0.55，t-SNE ARI = 0.29。PHATE 降维后聚类的结构保真度近 2 倍。

### 8.2 Tortuosity（轨迹弯曲度）

自计算：总轨迹长度 ÷ 起点到终点的直线距离。

```
Tortuosity = Σᵢ ‖point[i+1] - point[i]‖ / ‖point[-1] - point[0]‖
```

- 轨迹沿句子顺序连接相邻点
- 值越小 → 越像直线 → 逻辑推进越聚焦
- 值越大 → 绕来绕去 → 可能跑题或跳跃

**我的数据**：PHATE = 82.5，比 t-SNE (99.8) 和 PCA (875.9) 都低。

### 8.3 Trustworthiness（值得信赖度）

衡量降维后**近邻关系**的保留程度。

```
T(k) = 1 - (2 / (nk(2n-3k-1))) × Σᵢ Σⱼ (r(i,j) - k)
```

其中 r(i,j) 是低维空间中第 j 近的点在高维空间的排名。

值越高 → 低维空间中的近邻在高维空间中也确实近。t-SNE 此项最高（0.95），因为它专门优化局部结构。

### 8.4 Spearman r（连贯性排名保序）

降维前后的连贯性排名一致性。Spearman 秩相关系数：

```
ρ = 1 - (6 × Σ dᵢ²) / (n(n²-1))
```

其中 dᵢ = 降维前第 i 个连贯性的排名 - 降维后第 i 个连贯性的排名。

衡量的是"降维后连贯性的相对大小顺序是否保持不变"。

## 9. L2 欧氏距离

### 9.1 数学定义

两个向量 A 和 B 的 L2 距离（欧氏距离）：

```
‖A - B‖₂ = √(Σ (aᵢ - bᵢ)²)
```

### 9.2 在我的项目中的作用

**PELT 变点检测**：

PELT 在原始 embedding 上用 L2 距离检测变点。L2 距离对向量的**绝对位置变化**敏感——既包括方向变化（语义转折），也包括模长变化（信息密度变化）。

**Tortuosity 计算**：PHATE 输出的是欧氏空间坐标，计算相邻点之间的距离直接使用 L2。

### 9.3 余弦距离 vs L2 距离的选择

| 场景 | 选择 | 原因 |
|------|------|------|
| BGE-M3 embedding 相似度 | **余弦** | 方向承载语义，模长受句子长度影响 |
| PHATE knn 距离 | **余弦** | 同上，高维 embedding 统一用余弦 |
| PELT 变点检测 | **L2** | 需要检测绝对位置突变（方向+模长） |
| PHATE 降维后的轨迹 | **L2** | 低维欧氏坐标，L2 是自然选择 |
